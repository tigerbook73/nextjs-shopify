# /adr — 创建架构决策记录

用法：`/adr <决策标题>`

## 步骤 1：收集信息

如果用户已在命令中提供标题，跳过询问标题。
向用户收集以下信息（可以一次性询问）：

1. **背景**：为什么需要做这个决策？触发这个决策的上下文是什么？
2. **决策**：最终选择了什么？
3. **考虑过哪些替代方案**？各自的优缺点是什么？
4. **后果**：这个决策带来了哪些正面影响？哪些约束？

## 步骤 2：确定编号

读取 `docs/adr/README.md` 的决策索引表，找到当前最大编号，新 ADR 编号 = 最大编号 + 1。

## 步骤 3：生成文件

文件路径：`docs/adr/<四位编号>-<kebab-case-标题>.md`

按以下模板创建：

```markdown
# ADR-XXXX：<标题>

**状态**：已采纳
**日期**：<当前年月，格式 YYYY-MM>

## 背景

<用户提供的背景信息>

## 决策

<用户提供的决策内容，一句话说清楚>

## 考虑过的替代方案

| 方案 | 优点 | 缺点 / 排除原因 |
|------|------|---------------|
| <方案 A> | <优点> | <缺点> |
| <方案 B> | <优点> | <缺点> |

## 后果

### 正面影响
- <影响 1>

### 负面影响 / 约束
- <约束 1>

### 相关规范
- 见 `docs/conventions/architecture.md`（如适用）
```

## 步骤 4：更新索引

在 `docs/adr/README.md` 的决策索引表中追加新条目：

```
| [ADR-XXXX](XXXX-xxx.md) | <标题> | 已采纳 | <年月> |
```

## 步骤 5：输出确认

告知用户：
- 创建的文件路径
- 下一步是否需要同步更新 `docs/conventions/` 中的相关规范？
