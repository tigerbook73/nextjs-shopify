# /adr — 创建架构决策记录

用法：`/adr <决策标题>`

## 步骤 1：收集信息

如果用户已在命令中提供标题，跳过询问标题。向用户收集：

1. **背景**：为什么需要做这个决策？触发上下文是什么？
2. **决策**：最终选择了什么？
3. **替代方案**：考虑过哪些方案？各自优缺点？
4. **后果**：正面影响？带来哪些约束？

## 步骤 2：确定编号

读取 `docs/adr/README.md` 决策索引表，找到当前最大编号，新编号 = 最大编号 + 1。

## 步骤 3：生成文件

文件路径：`docs/adr/<四位编号>-<kebab-case-标题>.md`

```markdown
# ADR-XXXX：<标题>

**状态**：已采纳
**日期**：<YYYY-MM>
**影响范围**：全局

## 背景

<为什么需要做这个决策>

## 决策

<最终选择及理由，一句话说清楚>

## 考虑过的替代方案

| 方案     | 优点 | 缺点 / 排除原因 |
| -------- | ---- | --------------- |
| <方案 A> | ...  | ...             |

## 后果

### 正面影响

- ...

### 负面影响 / 约束

- ...
```

## 步骤 4：更新索引

在 `docs/adr/README.md` 决策索引表追加新条目：

```
| [ADR-XXXX](XXXX-xxx.md) | <标题> | 已采纳 | <YYYY-MM> |
```

## 步骤 5：输出确认

告知用户创建的文件路径，并询问是否需要同步更新 `docs/conventions/` 中的相关规范。
