# /learn-phase — Phase 代码分组讲解

对当前 Phase 已实现的代码进行分组讲解，帮助理解设计决策和实现细节。

**用法**：

- `/learn-phase`：从 G1 开始完整讲解
- `/learn-phase G3`：从指定组开始（用于续讲）

---

## 步骤 1：恢复上下文

找到当前进行中的 Phase 文档目录（`docs/features/` 下**不带** `-` 前缀的目录）：

1. 读 `PROGRESS.md`：了解交付物清单和备注
2. 读 `DESIGN.md`：了解架构决策和模块分组

如果用户指定了起始组（如 `G3`），跳过编号更小的组。

---

## 步骤 2：确定分组

优先使用 `DESIGN.md` 中已有的分组。如果文档未明确分组，按以下层次从下往上组织，每组 2-4 个相关文件：

1. **基础设施层**：常量、类型定义、工具函数
2. **数据获取层**：GraphQL queries/mutations、Shopify 客户端函数
3. **服务端逻辑层**：Server Actions、Route Handlers
4. **组件层**：Server Components → Client Components（从外到内）
5. **路由层**：`page.tsx`、`loading.tsx`、`layout.tsx`

---

## 步骤 3：逐组讲解

**每组开始前**：先读完本组所有文件，再开始讲解。

**每组讲解包含**：

- **变更对比**：用 `// 之前` / `// 之后` 展示关键改动，让用户看到"从哪里来"
- **设计原因**：解释为什么这样设计，不只是描述做了什么
- **权衡取舍**：如有其他方案，说明为什么选这个
- **层间关系**：说明本组如何配合其他层工作
- **非显而易见的细节**：专门拎出来讲（如版本兼容问题、隐含约束等）

**讲解节奏**：

- 每讲完一组，停下来输出：**G{N} 讲完，继续 G{N+1}？**
- 等用户确认后再继续，不要一次性讲完所有组
- 用户的追问是讲解的一部分，充分展开回答后再问是否继续

---

## 步骤 4：全部讲完后

输出本 Phase 的学习小结：

```
## Phase {N} 学习小结

| 组 | 文件 | 核心概念 |
|---|---|---|
| G1 | ... | ... |
...

**本 Phase 引入的新概念**：<列举>
**与上一 Phase 的关联**：<说明>
```

然后询问是否需要更新学习进度记录（memory）。
