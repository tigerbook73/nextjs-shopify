# ADR-0004：使用 shadcn/ui（@base-ui/react）作为 UI 组件库

**状态**：已采纳
**日期**：2026-05
**影响范围**：全局

## 背景

项目需要一套 UI 组件库来提供基础交互组件（按钮、对话框、下拉菜单等），同时要与 Tailwind CSS v4 兼容，并允许完全控制组件样式和行为，避免因第三方样式系统引入复杂的覆盖层。

## 决策

使用 shadcn/ui 方案，底层实现选择 `@base-ui/react`（Radix 的继任者），通过 Tailwind CSS 驱动样式，组件代码直接复制进项目而非作为黑盒依赖。

## 考虑过的替代方案

| 方案             | 优点                       | 缺点 / 排除原因                                         |
| ---------------- | -------------------------- | ------------------------------------------------------- |
| MUI / Ant Design | 组件丰富，文档完善         | 自带样式系统与 Tailwind 冲突，bundle 体积大，定制成本高 |
| Headless UI      | 无样式，与 Tailwind 配合好 | 组件数量少，不如 shadcn/ui 生态完整                     |
| 纯手写组件       | 完全自主                   | 开发成本高，可访问性（a11y）难以保证                    |

## 后果

### 正面影响

- 组件代码在项目内，可随意修改，无版本锁定问题
- 与 Tailwind CSS v4 完全兼容
- 可访问性由 @base-ui/react 底层保证，无需自己处理 ARIA
- 按需添加组件，不引入未用的代码

### 负面影响 / 约束

- 组件升级需要手动追踪 shadcn/ui 上游变更
- `src/components/ui/` 目录中的文件由 shadcn CLI 生成，不应手动修改原始文件（应通过覆盖层扩展）
- 相关规范：见 `docs/conventions/architecture.md` 技术栈表格
