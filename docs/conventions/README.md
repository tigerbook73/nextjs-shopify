# 规范文档索引

本目录是 nextjs-shoptify 所有编码规范的唯一权威来源。
AI 工具和开发者在实现任何功能前，应先阅读相关规范文件。

## 规范文件列表

| 文件 | 内容 |
| ---- | ---- |
| `architecture.md` | 技术选型、架构分层、禁止事项、引入新依赖的规则 |
| `coding.md` | TypeScript 规范、命名约定、注释规则、组件边界、提交规范 |
| `testing.md` | 测试定位、测什么/不测什么、工具、文件组织 |
| `directory.md` | 目录树、文件命名规则、新增文件决策树 |
| `graphql.md` | 查询/变更/Fragment 规范、错误处理、分页、缓存标签 |
| `ai-workflow.md` | AI 工具选择矩阵、开发流程、规范冲突决策框架 |

架构决策背景记录：`../adr/`

## 规范更新流程

1. 发现规范与实现不符时，在 Claude Code 中运行 `/update-convention`
2. 判断冲突类型（实现不符、规范过时、新场景）
3. 重大变更需同步创建 ADR（运行 `/adr`）
4. 更新规范文件并提交

## 健康检查

定期运行 `/check-conventions`，确认近期改动与规范保持一致。
