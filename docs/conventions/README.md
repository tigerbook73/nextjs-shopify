AI 工具（Claude Code、Codex）在实现功能前应先阅读相关规范文件。

## 规范更新流程

规范不是一成不变的。当出现以下情况时，应更新规范而非绕过它：

### 什么时候应该更新规范

- 新引入的框架/库带来了与现有规范不兼容的模式
- 现有规范在某个场景下无法合理应用
- 实践中发现规范的某条规则导致了更多问题而非解决问题

### 什么时候不应该更新规范

- 只是「这次懒得遵守」
- 一次性的特殊情况（应在代码中注释说明，不改规范）
- 还没经过实践验证的想法

### 更新步骤

1. **识别冲突**：明确说明哪条规范、与什么新需求冲突
2. **提案**：说明新规范应该怎么写，以及影响范围
3. **创建 ADR**：用 `/adr` 命令记录决策背景和后果（见 `../adr/README.md`）
4. **更新规范文件**：修改对应的 `*.md`，同步更新本文件的描述（如需要）
5. **通知 AI 工具**：若涉及 CLAUDE.md 或 AGENTS.md 的引用，同步更新

> 使用 Claude Code 的 `/update-convention` 命令可以引导完成以上全流程。

---

## 规范健康检查

定期或在每个 Phase 结束时，用 `/check-conventions` 命令审查近期代码
是否出现规范偏离，以便及时修正或更新规范。
